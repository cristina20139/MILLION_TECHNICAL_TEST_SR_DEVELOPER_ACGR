🧪 PRUEBA TÉCNICA - DESARROLLADOR SENIOR FULLSTACK

1️⃣ Resumen del Proyecto:

🏢 Una gran empresa inmobiliaria requiere la creación de una API para obtener información sobre propiedades almacenadas en una base de datos, y una página web para mostrar esta información. El objetivo es crear una aplicación full-stack utilizando .NET, MongoDB, C# y ReactJS o Next.js. El proyecto implicará construir un conjunto de servicios para gestionar datos de propiedades y crear una interfaz de usuario para mostrarlos.

2️⃣ Tecnologías Requeridas:

  • 2.1. 🖥️ Backend: .NET 8 o 9, C#
  • 2.2. 🗄️ Base de datos: MongoDB
  • 2.3. 🌐 Frontend: ReactJS o Next.js
  • 2.4. 🧪 Pruebas: NUnit para pruebas unitarias

3️⃣ Desglose de Tareas:

3.1 🚀 Desarrollo Backend (API):

  • 3.1.1. Crear una API en C# utilizando .NET 8 o 9 para recuperar datos de propiedades desde la base de datos MongoDB.
  • 3.1.2. Implementar filtros en la API para obtener una lista de propiedades según parámetros como nombre, dirección y rango de precios.
  • 3.1.3. Definir los campos de los DTOs, tales como:
    • 🆔 IdOwner
    • 🧾 Name
    • 📍 AddressProperty
    • 💲 PriceProperty
    • 🖼️ Una sola imagen

3.2 💻 Desarrollo Frontend (Página Web):

3.2.1. Crear una página web utilizando ReactJS o Next.js para mostrar los datos de propiedades.

3.2.2. La página debe incluir:

    📋 Una lista de propiedades, obtenida desde la API.

    🔍 Filtros para buscar propiedades (por nombre, dirección y rango de precios).

    👁️‍🗨️ Opción para ver más detalles sobre propiedades individuales.

📱3.2.3. Asegurar que el frontend sea responsive, brindando una experiencia de usuario fluida en diferentes dispositivos.

4️⃣ Criterios de Evaluación:

🧱 4.1. Arquitectura Backend y Frontend: Implementar una arquitectura limpia y eficiente para la API backend y la página web frontend.

📁 4.2. Estructura del Código: Organizar el código de manera modular y mantenible.

📝 4.3. Documentación: Proporcionar documentación clara y concisa tanto para la API como para el código frontend.

✅ 4.4. Buenas Prácticas: Seguir buenas prácticas en el desarrollo backend y frontend, incluyendo:

    📐 4.4.1. Arquitectura limpia

    ⚠️ 4.4.2. Manejo adecuado de errores

    ⚡ 4.4.3. Consultas optimizadas a la base de datos (si es necesario)

🚀 4.5. Rendimiento: Asegurar que tanto la API como el frontend estén optimizados para el rendimiento, especialmente al manejar grandes volúmenes de datos o múltiples filtros.

🔬 4.6. Pruebas Unitarias: Escribir pruebas unitarias para la API backend y los componentes frontend utilizando NUnit u otros frameworks adecuados.

🧼 4.7. Código Limpio: Asegurar que el código sea legible, mantenible y que siga las convenciones de codificación establecidas.

✅ Al completar este proyecto, demostrarás tu capacidad para construir una aplicación full-stack con una arquitectura limpia, buen rendimiento y pruebas y documentación adecuadas.
