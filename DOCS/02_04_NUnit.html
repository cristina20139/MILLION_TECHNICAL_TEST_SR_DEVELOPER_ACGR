¿Qué es NUnit?

NUnit es un framework de pruebas unitarias para .NET. Permite a los desarrolladores escribir y ejecutar pruebas automatizadas para verificar que su código funcione correctamente. Es uno de los frameworks más utilizados en el ecosistema .NET, similar a JUnit en Java.

Arquitectura de NUnit

La arquitectura de NUnit se basa en los siguientes componentes clave:

  • Core (núcleo): Contiene las clases base necesarias para definir y ejecutar pruebas.
  • Test Runner: Ejecuta las pruebas y muestra los resultados.
  • Assertion Library: Proporciona métodos para verificar condiciones (Assert.AreEqual(), etc.).
  • Attributes: Utiliza anotaciones como [Test], [SetUp] para marcar métodos especiales.
  • Extensibilidad: Permite crear extensiones personalizadas para adaptar los resultados, fixtures, etc.

Características Principales

  • Soporte para pruebas unitarias, de integración y de regresión.
  • Ejecuta pruebas en paralelo.
  • Compatibilidad con .NET Core y .NET Framework.
  • Compatible con runners como Visual Studio Test Explorer o línea de comandos.

Ejemplo Básico

A continuación, un ejemplo básico de prueba con NUnit:

using NUnit.Framework;

namespace Calculadora.Tests
{
    public class CalculadoraTests
    {
        [SetUp]
        public void Setup()
        {
            // Código que se ejecuta antes de cada prueba
        }

        [Test]
        public void Sumar_DosNumeros_RetornaResultadoCorrecto()
        {
            int resultado = 2 + 3;
            Assert.AreEqual(5, resultado);
        }
    }
}

Atributos Comunes

  • [Test]: Marca un método como prueba.
  • [SetUp]: Se ejecuta antes de cada prueba.
  • [TearDown]: Se ejecuta después de cada prueba.
  • [TestCase]: Permite pruebas parametrizadas.
  • [Ignore]: Ignora una prueba específica.

Ejemplo con Parámetros

[TestCase(2, 3, 5)]
[TestCase(-1, -1, -2)]
public void Sumar_Parametros_RetornaResultadoCorrecto(int a, int b, int esperado)
{
    Assert.AreEqual(esperado, a + b);
}
